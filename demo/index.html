Chinese Palette

A general website theme with a highlight.js adapter

The refactor keeps the original Base16 color relationships, but exposes them through reusable semantic tokens for pages, forms, tables, and code.

Base16 reference highlight.js

Semantic Tokens

Use --color-bg, --color-surface, and --color-text for document styling, then layer on syntax colors only where needed.

  • Accent
    var(--base08)
  • Success
    var(--base0B)
  • Link
    var(--base0D)
  • Keyword
    var(--base0E)

Prose and Tables

The theme is aimed at long-form content, so links, inline emphasis, blockquotes, tables, and inline code all share the same token system.

The original project was only a syntax theme. The refactor keeps that behavior, but makes the palette useful for the rest of the page too.
Token Purpose
--color-surface Cards, buttons, and code blocks
--color-border Subtle separators and table lines
--syntax-string Reusable token for code adapters
Semantic aliases keep the palette reusable outside syntax highlighting.

Code Sample

<!doctype html>
<html data-theme="chinese-palette">
  <link rel="stylesheet" href="chinese-palette.css">
  <main>
    <code>semantic tokens plus syntax colors</code>
  </main>
</html>

Inline code still reads cleanly, for example --color-code-bg and --syntax-keyword.

Keyboard hints also inherit the palette: Cmd + K.
